SQL Task 7: Window Functions and Ranking

1. Row Number (Ranking Employees)

Command:
SELECT Name, Department, Salary,
ROW_NUMBER() OVER (ORDER BY Salary DESC) AS RowNum
FROM Employees;

Explanation:
Assigns a unique row number based on salary (highest first).

---

2. Rank Employees by Salary

Command:
SELECT Name, Department, Salary,
RANK() OVER (ORDER BY Salary DESC) AS RankNum
FROM Employees;

Explanation:

* Same salary → same rank
* Skips rank numbers (e.g., 1,1,3)

---

3. Dense Rank

Command:
SELECT Name, Department, Salary,
DENSE_RANK() OVER (ORDER BY Salary DESC) AS DenseRank
FROM Employees;

Explanation:

* Same salary → same rank
* No gaps (e.g., 1,1,2)

---

4. Partition by Department

Command:
SELECT Name, Department, Salary,
RANK() OVER (PARTITION BY Department ORDER BY Salary DESC) AS DeptRank
FROM Employees;

Explanation:

* Resets ranking for each department
* Independent ranking groups

---

5. LAG (Previous Row Value)

Command:
SELECT Name, Salary,
LAG(Salary) OVER (ORDER BY Salary DESC) AS PreviousSalary
FROM Employees;

Explanation:
Shows previous row’s salary.

---

6. LEAD (Next Row Value)

Command:
SELECT Name, Salary,
LEAD(Salary) OVER (ORDER BY Salary DESC) AS NextSalary
FROM Employees;

Explanation:
Shows next row’s salary.

---

7. Salary Difference Using LAG

Command:
SELECT Name, Salary,
Salary - LAG(Salary) OVER (ORDER BY Salary DESC) AS SalaryDiff
FROM Employees;

Explanation:
Calculates difference with previous employee salary.

---

KEY CONCEPT

OVER() defines window:

* ORDER BY → sorting
* PARTITION BY → grouping

---

ROW_NUMBER vs RANK vs DENSE_RANK

ROW_NUMBER:

* Always unique

RANK:

* Skips numbers on ties

DENSE_RANK:

* No gaps in ranking

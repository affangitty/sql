SQL Task 9: Stored Procedures and User-Defined Functions

1. Create Sales Table

Command:
CREATE TABLE Sales (
Id INT PRIMARY KEY,
Amount DECIMAL(10,2),
SaleDate DATE
);

---

2. Insert Sample Data

Command:
INSERT INTO Sales VALUES
(1, 5000, '2024-01-01'),
(2, 7000, '2024-01-10'),
(3, 3000, '2024-02-01');

---

3. Create Stored Procedure

Command:
CREATE PROCEDURE GetTotalSales
@StartDate DATE,
@EndDate DATE
AS
BEGIN
SELECT SUM(Amount) AS TotalSales
FROM Sales
WHERE SaleDate BETWEEN @StartDate AND @EndDate;
END;

Explanation:

* Takes date range as input
* Returns total sales within that range

---

4. Execute Stored Procedure

Command:
EXEC GetTotalSales '2024-01-01', '2024-01-31';

Explanation:
Returns total sales for January.

---

5. Create Scalar Function (Discount)

Command:
CREATE FUNCTION CalculateDiscount (@Amount DECIMAL(10,2))
RETURNS DECIMAL(10,2)
AS
BEGIN
RETURN @Amount * 0.1;
END;

Explanation:
Returns 10% discount on amount.

---

6. Use Function

Command:
SELECT Id, Amount,
dbo.CalculateDiscount(Amount) AS Discount
FROM Sales;

Explanation:
Applies function to each row.

---

7. Table-Valued Function (Optional)

Command:
CREATE FUNCTION GetHighSales (@MinAmount DECIMAL(10,2))
RETURNS TABLE
AS
RETURN (
SELECT * FROM Sales
WHERE Amount > @MinAmount
);

---

8. Use Table Function

Command:
SELECT * FROM GetHighSales(4000);

Explanation:
Returns sales above given amount.

---

KEY DIFFERENCE

Stored Procedure:

* Can perform multiple operations
* Can modify data
* Called using EXEC

Function:

* Must return value
* Used inside SELECT
* Cannot modify data

---

WHEN TO USE

Stored Procedure:

* Business workflows
* Complex operations

Function:

* Calculations
* Reusable expressions
